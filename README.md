#### Обоснование выбора набора данных для задачи классификации:  
Для задачи классификации используется набор данных Student Alcohol Consumption, который позволяет предсказать, сдаст ли студент итоговый экзамен. Этот выбор обоснован важностью прогнозирования академической успеваемости в образовательных учреждениях: модель поможет выявлять студентов в группе риска для своевременной поддержки, анализировать влияние социальных факторов на успехи в учебе и оптимизировать учебный процесс. #### Обоснование выбора набора данных для задачи регрессии:  
Для задачи регрессии выбран датасет House Rent Prediction, который ставит целью прогнозирование ежемесячной арендной платы. Эта задача крайне востребована на рынке недвижимости: автоматическая оценка стоимости аренды полезна как для арендодателей и риелторов, так и для арендаторов, желающих проверить справедливость цены. #### Выбор метрик качества и их обоснование:  
Метрики качества для задачи классификации:  
Accuracy (точность) - основная метрика через knn.score() и accuracy_score  
Precision, Recall, F1-score - вычисляются через precision_score(), recall_score(), f1_score()  
Confusion Matrix (матрица ошибок) - визуализация через confusion_matrix()  
Classification Report - детальный отчет по классам  

Обоснование: в задаче классификации успеваемости студентов основными метриками выбраны Accuracy, Precision, Recall и F1-score, что полностью соответствует характеру задачи прогнозирования сдачи экзамена. Accuracy предоставляет общую оценку точности модели, но ключевое значение имеет F1-score — гармоническое среднее между Precision и Recall, которое особенно важно при возможном дисбалансе классов (успевающих и неуспевающих студентов). Высокий Recall критичен для минимизации пропуска студентов из группы риска (False Negative), в то время как Precision важен для избежания ложных тревог (False Positive). Дополнительно матрица ошибок (Confusion Matrix) наглядно показывает распределение ошибок по типам, что позволяет анализировать практические последствия ошибочных прогнозов для образовательного процесса  


Метрики качества для задачи регрессии: 
R² Score (коэффициент детерминации) - основная метрика через knn.score()  
MSE (среднеквадратичная ошибка) - через mean_squared_error()  
RMSE (корень из MSE) - вычисляется как sqrt(MSE)  
Визуальный анализ - scatter plot предсказанных vs фактических значений  

Обоснование: для задачи регрессии прогнозирования арендной платы выбраны R² score, MSE и RMSE. R² score оптимален как основная метрика, поскольку показывает долю дисперсии цен, объясняемую моделью, что позволяет оценить её общую объяснительную силу. MSE и RMSE дополняют оценку, фокусируясь на величине ошибок: MSE строго штрафует за крупные отклонения (критически важно при работе с дорогим жильём), а RMSE, измеряемая в тех же единицах, что и целевая переменная (рубли), даёт интуитивно понятную оценку средней ошибки в денежном выражении. Визуализация предсказаний против фактических значений позволяет качественно оценить характер ошибок и выявить систематические смещения. Такой набор метрик обеспечивает комплексную оценку как объяснительной способности модели, так и практической точности прогнозов в контексте рынка недвижимости
